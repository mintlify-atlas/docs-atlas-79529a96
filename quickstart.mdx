This guide will help you get DevPush installed and deploy your first application. By the end, you'll have a working deployment platform running on your own server.

<Warning>
  Supported on Ubuntu 20.04+ and Debian 11+. Other distributions may work but aren't officially supported.
</Warning>

## Prerequisites

Before you begin, ensure you have:

- A fresh Ubuntu 20.04+ or Debian 11+ server with root/sudo access
- A domain name (e.g., `example.com`) with DNS access
- A GitHub account
- An email service (Resend API key or SMTP credentials)

## Step 1: Install DevPush

Run the installation script on your server:

```bash
curl -fsSL https://install.devpu.sh | sudo bash
```

The installer will:
- Install Docker and required dependencies
- Create a system user (`devpush`)
- Clone the DevPush repository
- Generate secure secrets
- Set up systemd service

<Note>
  The installation script automatically detects the latest stable version and installs it. The process typically takes 2-3 minutes.
</Note>

## Step 2: Create a GitHub App

DevPush uses a GitHub App for authentication and repository access.

1. Follow the guide at [devpu.sh/docs/guides/github-app](https://devpu.sh/docs/guides/github-app)
2. Save the following credentials:
   - App ID
   - App Name
   - Private Key (PEM format)
   - Webhook Secret
   - Client ID
   - Client Secret

<Tip>
  Keep your GitHub App credentials in a secure location. You'll need them in the next step.
</Tip>

## Step 3: Configure Environment Variables

Edit the environment file with your configuration:

```bash
sudo nano /var/lib/devpush/.env
```

Fill in the required values:

<CodeGroup>
```bash Domains & SSL
# Your main domain for the DevPush application
APP_HOSTNAME=devpush.example.com

# Wildcard domain for deployments (e.g., *.deploy.example.com)
DEPLOY_DOMAIN=deploy.example.com

# Email for Let's Encrypt SSL certificate notifications
LE_EMAIL=admin@example.com
```

```bash GitHub App
# Credentials from Step 2
GITHUB_APP_ID=123456
GITHUB_APP_NAME=my-devpush-app
GITHUB_APP_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----\nYour\nPrivate\nKey\nHere\n-----END RSA PRIVATE KEY-----"
GITHUB_APP_WEBHOOK_SECRET=your-webhook-secret
GITHUB_APP_CLIENT_ID=Iv1.abc123def456
GITHUB_APP_CLIENT_SECRET=your-client-secret
```

```bash Email (Resend)
# If using Resend for email delivery
EMAIL_SENDER_ADDRESS=noreply@example.com
RESEND_API_KEY=re_abc123def456
```

```bash Email (SMTP)
# Alternative: Use SMTP instead of Resend
EMAIL_SENDER_ADDRESS=noreply@example.com
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USERNAME=your-smtp-username
SMTP_PASSWORD=your-smtp-password
```
</CodeGroup>

<Note>
  The installer automatically generates `SECRET_KEY`, `ENCRYPTION_KEY`, `POSTGRES_PASSWORD`, and `SERVER_IP`. You don't need to modify these values.
</Note>

## Step 4: Configure DNS

Set up two DNS A records pointing to your server's IP address:

<Steps>
  <Step title="App hostname">
    Create an A record for your main domain:
    ```
    A    devpush.example.com    →    YOUR_SERVER_IP
    ```
  </Step>
  
  <Step title="Wildcard deployment domain">
    Create a wildcard A record for deployments:
    ```
    A    *.deploy.example.com    →    YOUR_SERVER_IP
    ```
  </Step>
</Steps>

<Warning>
  Wait for DNS propagation before starting the service. This typically takes a few minutes but can take up to 24 hours.
</Warning>

## Step 5: Start DevPush

Start the DevPush service:

```bash
sudo systemctl start devpush.service
```

Check the status:

```bash
sudo systemctl status devpush.service
```

You should see "active (running)" in the output.

<Tip>
  You can view real-time logs with: `sudo journalctl -u devpush.service -f`
</Tip>

## Step 6: Access DevPush

Open your browser and navigate to your app hostname:

```
https://devpush.example.com
```

You should see the DevPush login page. Sign in with your GitHub account.

## Step 7: Deploy Your First App

<Steps>
  <Step title="Create a team">
    After logging in, create a team for organizing your projects.
  </Step>
  
  <Step title="Add a project">
    Click "New Project" and connect one of your GitHub repositories.
  </Step>
  
  <Step title="Configure environment">
    Set up your first environment (e.g., "production") and map it to a branch (e.g., "main").
  </Step>
  
  <Step title="Deploy">
    Push a commit to the mapped branch or click "Deploy" to trigger your first deployment.
  </Step>
  
  <Step title="Monitor">
    Watch the real-time build and deployment logs in the DevPush dashboard.
  </Step>
</Steps>

## Verification

To verify everything is working:

1. **Check service status**: `sudo systemctl status devpush.service`
2. **View logs**: `sudo journalctl -u devpush.service -f`
3. **Access the dashboard**: Visit your `APP_HOSTNAME` in a browser
4. **Check SSL certificate**: Ensure you see a valid Let's Encrypt certificate

## Common Issues

<AccordionGroup>
  <Accordion title="Service won't start">
    Check the logs for errors:
    ```bash
    sudo journalctl -u devpush.service -n 50
    ```
    Common causes:
    - Missing or invalid environment variables in `/var/lib/devpush/.env`
    - DNS not yet propagated
    - Port 80 or 443 already in use
  </Accordion>
  
  <Accordion title="Can't access the dashboard">
    - Verify DNS is pointing to your server: `dig +short devpush.example.com`
    - Check firewall rules allow ports 80 and 443
    - Ensure the service is running: `sudo systemctl status devpush.service`
  </Accordion>
  
  <Accordion title="GitHub login fails">
    - Verify GitHub App credentials in `/var/lib/devpush/.env`
    - Check that the GitHub App callback URL matches your `APP_HOSTNAME`
    - Ensure the GitHub App is installed on your account/organization
  </Accordion>
  
  <Accordion title="SSL certificate issues">
    - Verify DNS is propagated: `dig +short devpush.example.com`
    - Check Let's Encrypt rate limits
    - Ensure `LE_EMAIL` is set correctly
    - View Traefik logs: `sudo /opt/devpush/scripts/compose.sh logs traefik`
  </Accordion>
</AccordionGroup>
